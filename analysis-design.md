# 📑 Analysis and Design Document

## 📚 ระบบร้านขายหนังสือ (Book Store Management System)

---

# 📌 1. Problem Statement | คำอธิบายปัญหา

ปัจจุบันร้านขายหนังสือแบบดั้งเดิมยังมีข้อจำกัดในการให้บริการลูกค้า
เช่น การค้นหาหนังสือ การตรวจสอบจำนวนสินค้าคงเหลือ
การสั่งซื้อ การชำระเงิน และการติดตามสถานะคำสั่งซื้อ
ทำให้ลูกค้าเสียเวลาและผู้ดูแลร้านต้องจัดการข้อมูลด้วยตนเอง

นอกจากนี้การจัดการสต็อกสินค้า
การตรวจสอบหลักฐานการชำระเงิน
และการสรุปยอดขายยังไม่มีระบบช่วยสนับสนุน
ส่งผลให้เกิดข้อผิดพลาดในการดำเนินงาน
และใช้เวลามากในการบริหารจัดการ

โครงงานนี้จึงพัฒนาระบบร้านขายหนังสือออนไลน์
เพื่อให้ลูกค้า พนักงาน และผู้ดูแลระบบ
สามารถใช้งานผ่านเว็บไซต์ได้อย่างสะดวก
พร้อมช่วยเพิ่มประสิทธิภาพในการบริหารจัดการร้านค้า

---

# 🔍 2. Pain Points

## 👤 ฝั่งผู้เยี่ยมชมและลูกค้า (Visitor & Customer)

| ปัญหา | ความรุนแรง | ผลกระทบ |
|---------|-----------|-----------|
| ค้นหาหนังสือได้ยาก | สูง | ใช้เวลานาน |
| ไม่ทราบจำนวนสินค้าคงเหลือ | สูง | สั่งซื้อไม่ได้ |
| ไม่มีตัวช่วยแนะนำหนังสือ | กลาง | ตัดสินใจซื้อยาก |
| ไม่สามารถติดตามสถานะคำสั่งซื้อ | สูง | ลูกค้าไม่มั่นใจ |
| การชำระเงินและแนบสลิปไม่มีระบบ | สูง | ข้อมูลผิดพลาด |

---

## 👨‍💼 ฝั่งพนักงาน (Staff)

| ปัญหา | ความรุนแรง | ผลกระทบ |
|---------|-----------|-----------|
| ตรวจสอบคำสั่งซื้อช้า | สูง | จัดส่งล่าช้า |
| ตรวจสอบหลักฐานการชำระเงินด้วยตนเอง | สูง | เกิดข้อผิดพลาด |
| ไม่มีระบบแจ้งเตือนคำสั่งซื้อใหม่ | สูง | พลาดการดำเนินงาน |
| อัปเดตสถานะการจัดส่งไม่สะดวก | กลาง | ลูกค้าได้รับข้อมูลไม่ถูกต้อง |

---

## 🔐 ฝั่งผู้ดูแลระบบ (Admin)

| ปัญหา | ความรุนแรง | ผลกระทบ |
|---------|-----------|-----------|
| จัดการสินค้าและสต็อกยาก | สูง | ข้อมูลคลาดเคลื่อน |
| ไม่มีระบบแจ้งเตือนสินค้าใกล้หมด | สูง | สินค้าหมดสต็อก |
| ไม่มี Dashboard วิเคราะห์ข้อมูล | สูง | ตัดสินใจยาก |
| ตรวจสอบการทำงานของพนักงานยาก | กลาง | ตรวจสอบย้อนหลังไม่ได้ |

# 🔒 4. Non-Functional Requirements | ความต้องการที่ไม่ใช่หน้าที่

| NFR ID | หมวดหมู่ | รายละเอียด |
|----------|-----------|-------------|
| NFR-01 | Performance | ระบบตอบสนองภายใน 3 วินาที |
| NFR-02 | Usability | ใช้งานง่าย ไม่ซับซ้อน |
| NFR-03 | Security | กำหนดสิทธิ์การใช้งานตามบทบาท Visitor / Customer / Staff / Admin |
| NFR-04 | Reliability | ระบบทำงานได้ต่อเนื่องและลดข้อผิดพลาดของข้อมูล |
| NFR-05 | Availability | ระบบพร้อมให้บริการอย่างน้อย 95% |
| NFR-06 | Scalability | รองรับการเพิ่มจำนวนหนังสือและผู้ใช้งาน |
| NFR-07 | Maintainability | ใช้ Git และ GitHub ในการควบคุมเวอร์ชัน |
| NFR-08 | Compatibility | รองรับ Chrome, Edge, Firefox และ Safari |
| NFR-09 | Data Integrity | ข้อมูลสินค้า คำสั่งซื้อ และสต็อกต้องถูกต้องตรงกัน |
| NFR-10 | Backup | รองรับการสำรองข้อมูลเพื่อป้องกันข้อมูลสูญหาย |

# 📌 สรุป

เอกสารฉบับนี้ใช้เป็นแนวทางในการวิเคราะห์และออกแบบระบบร้านขายหนังสือออนไลน์
โดยครอบคลุมปัญหาของระบบเดิม ความต้องการของผู้ใช้งาน
และข้อกำหนดต่าง ๆ ของระบบ
เพื่อใช้ประกอบการออกแบบ Diagram ฐานข้อมูล API
